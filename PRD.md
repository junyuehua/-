# PRD — 清明上河图互动Web App

版本：v1（首次完整整理） | 关联文档：`CLAUDE.md`（现状速览）、`decisions-log.md`（决策推理过程）

---

## 1. 产品概述

一个让观者与高清《清明上河图》（故宫博物院藏，北宋张择端本）互动的网页应用。观者可以放大画作欣赏细节，并与预先标注好的画面元素互动，学习画中描绘的北宋人文历史信息。

产品哲学：**画面为主，UI服务于画面。** 所有壳层控件的设计都要服务于"看画"这个核心体验，不能喧宾夺主。

原作实物尺寸（故宫博物院官方数据）：绢本，淡设色，纵24.8厘米，横528厘米。

---

## 2. 核心目标（两大主功能）

1. **放大画作**，仔细欣赏笔触与细节
2. **与预标注的画面元素互动**（分类：建筑 / 人物 / 物件 / 植物 / 动物 / 印章），了解对应的历史信息

---

## 3. MVP 范围

### 3.1 核心浏览体验
- 主画布支持拖拽平移、滚轮缩放
- 缩放范围【2026-07-10 改版，废弃旧"锁定最小100%（扫描分辨率）"规则】：最小 = **整卷完整入画**（fit 全览，随视口尺寸动态计算），画外露出的部分显示莲花纹渐变背景；最大暂定为扫描分辨率的500%，等瓦片架构最高精度层明确后再定
- **比例基准 = 实物比例**：100% = 画在屏幕上呈现真实物理大小（按画心高24.8cm与CSS 96dpi折算，≈扫描分辨率的12.3%）——产品目的是让用户直观感受这幅画本来的大小，从而体会细节刻画之精妙。物理精度受限于浏览器拿不到真实屏幕DPI，按CSS标准折算即可
- 初始视图 = 卷首（内容最右端）+ 实物比例100%，如同站在真迹前展开画卷
- 不做阻塞式整图加载，采用深度缩放瓦片技术（类似OpenSeadragon/IIIF）：先加载低分辨率全览图，高清瓦片按视口和缩放级别按需流式加载；加载进度条含义 = "当前视口清晰度"，不是"整图加载进度"

### 3.2 观看模式（三选一，可切换）

【2026-07-11 由两模式扩为**三模式**。核心架构原则见下方"镜头持续"。旧文档里"沉浸模式=卧游=UI全淡出"的两模式结构已被本节取代——原先那个"UI全隐、纯欣赏"的模式现拆分并升级为下面的**神游**，"卧游"则重新定义为带标记的自动展卷。】

**核心架构原则（镜头持续）：镜头（位置 + 缩放）是持续存在的共享层，切换模式永远不搬动镜头**——只改变"在镜头上发生什么"（自动平移开不开、UI/标记显不显示）。"回到卷首"只发生在两处：app 首次加载（初始视图 = 卷首 + 实物100%），或用户明确点「重播」。任何模式切换都从当前位置/缩放接着来，绝不跳。

**① 读画模式（learn，默认打开）** —— 主动 + 有信息
UI 常驻可见，画面静止、由用户自己拖拽/缩放探索；标记可见可交互，悬停出信息卡。即现有这套。

**② 卧游模式** —— 被动可交互 + 有信息（"卧以游之"，2026-07-11 重新定义；2026-07-12 壳层改为与神游一致全隐）
画卷自动从**卷首(右)向卷尾(左)缓慢平移**，标记全部在；**壳层与神游一样全部淡出**（2026-07-12 改，取代首版"壳层常驻"——卧游与神游的区别只剩"标记显不显示"一件事），靠上下边缘热区唤出（同 §3.6 神游那套，含武装机制）。规则：
- 用户仍可正常缩放；**缩放不打断平移**（持续移动）。平移锁**屏幕速度恒定**（非内容速度）——故放大后在卷上的推进自然变慢（贴近细看时多停留），视觉节奏始终一致，避免高缩放下画面飞驰致晕。
- **悬停任一标记 → 平移缓停**（非硬刹），信息卡浮现；鼠标移开 + 卡片消失（约 250–300ms 缓冲，复用悬停卡逻辑）→ **平移缓起、继续向左**。
- 用户可**手动拖拽当快进/倒带**（左右皆可 scrub）；松手后自动续播，方向恒为向左。
- 进入卧游 = 从**当前位置 + 当前缩放**开始向左，**不回卷首**。
- 走到卷尾停住、不循环（回看靠拖拽倒带）。【2026-07-12 砍掉「重播」常驻按钮——首版已实现（回卷首+复位实物100%+快速淡切）后用户拍板去掉，不再保留任何重播入口】

**③ 神游模式** —— 被动 + 无信息（命名取自与"卧游"同源的山水鉴赏传统："卧游"身体躺着游、"神游"精神游进画里，更进一层）
运动逻辑与卧游**完全一致**（自动向左平移 + 可缩放 + 可拖拽），但**所有 UI 全部消失**：无标记、无壳层、无控件，只剩画。因无标记，故无"悬停暂停"。退出/切模式靠"鼠标移到边缘唤出控件"或键盘（见 §3.6/§3.7）。

**三者关系：** 一条从主动到忘我的谱系——读画(主动+信息) → 卧游(被动可交互+信息) → 神游(被动+纯画面)，UI 递减、沉浸递增。**卧游与神游共用同一台 auto-pan 引擎**，只差"是否显示标记"一个开关（2026-07-12 起壳层两者都隐、都走边缘唤出），别实现两遍。

**代码代号：** 现有内部代号 learn/immerse（两模式）需扩成三个——读画=learn 保留；卧游、神游各需代号（建议如 travel / immerse），旧 immerse（UI全隐）语义上最接近神游，卧游是全新的一档。

（**放弃的方案：** 曾设想"自动逐个弹开每张卡片"的导览式播放，后否——卧游改成"匀速流过 + 你悬停哪个读哪个"，保留用户对"读什么"的选择权、也不强制走完 684 点，更自然。见 `decisions-log.md`。）

### 3.3 卷轴呈现方式（MVP仅一种）
- MVP只做**完全展开**：连续展开的长卷视图，标注/hotspot互动在此模式下生效
- **分段臂展**（还原手卷阅读体验的替代格式）连同它的切换控件已**移出MVP**，见§7路线图；MVP阶段不存在格式切换，也就不需要对应的checkbox或`S`快捷键

### 3.4 导航与定位
- 位置指示组件：卷轴缩略图导航条，**通栏铺满整个屏幕宽度**（不做居中限宽）
- 可点击导航条任意位置跳转到对应卷轴位置

### 3.5 缩放控制
- 缩放百分比实时读数（比例toast，基准=实物比例，见§3.1）：缩放时出现，停留约1秒后自动消失（时长待实测调整），不常驻
- 一个快捷缩放按钮：**恢复实际大小**（跳转到实物比例100%），**不常驻**——只随比例toast一起出现/消失，且仅当当前比例≠100%时显示；行为上只改变缩放倍数，**保持当前观看位置的中心点不变**，不跳转到卷首等固定位置
- **"放大到最大"按钮已移出MVP**，见§7路线图——原因是它依赖深度缩放瓦片架构的最高精度层设计，该架构本身尚未详细设计，先把MVP范围收窄到"恢复实际大小"这一个明确的按钮

**单击放大：**
- 在画布空白处单击（非拖拽）= 在当前比例上放大50%（×1.5，2026-07-10 从×1.3上调），缩放中心精确锚定在点击的像素位置（复用滚轮缩放已验证的"锚定鼠标位置"变换数学），有200-300ms左右的缓动过渡动画，不是瞬间跳变
- **不做双击放大**——只有单击这一档，因此单击可以立即触发缩放，不需要延迟等待判断"是否会有第二次点击"
- 单击与拖拽平移的判定阈值建议比常规UI更宽松一点（约5-8px的移动容差），因为"误判成单击"的代价是一次意外缩放，比"误判成拖拽"的代价更高，需要更保守地偏向判定为拖拽
- **所有UI元素（marker、按钮、信息卡、导航条等）自身的点击都必须stopPropagation**，不能让点击事件冒泡到画布层触发这个单击放大——画布层的单击放大只对真正点在空白内容上的点击生效
- **marker点击 = 触发同样的放大动作，但缩放中心是marker的精确锚点坐标，不是原始鼠标点击位置**——比一般空白点击更精确，效果是"点一个知识点=精确凑近它"。这个动作叠加在悬停展示信息卡之上，不是替代关系（见3.8悬停交互）

### 3.6 UI 壳层

**顶部（元控件区）：**
- **读画 / 卧游 / 神游 三模式切换**（tab 并列按钮；2026-07-11 由两模式扩为三，见 §3.2）
- ~~双语切换toggle~~ **英文支持与语言切换按钮已移出 MVP（2026-07-11，见§7路线图）**；`*_en` 数据字段保留继续填
- 背景音乐开关（默认关闭，2026-07-11 落地）：素材 `app/public/music/bgm.m4a`（`Music/music.mp4` 转码 AAC 128k）；off 态用 Google `music_off` 图标、播放中 `music_note`；首次点击从头播放并循环，再点暂停，再点从暂停处**续播**（不回头）

**底部：**
- 通栏缩略图导航条（见3.4）
- 缩放按钮（恢复实际大小）+ 缩放百分比读数（见3.5），视觉上作为一个"缩放控制簇"整体呈现，与导航条区分开

**壳层可见性规则（顶部/底部共享同一套机制；2026-07-12 卧游从"常驻"档移入"淡出"档）：**
- **读画**：壳层默认常驻显示，不自动淡出（标记也可见）
- **卧游 / 神游**：壳层默认全部淡出（卧游**标记保留**、神游**连同标记**一起淡出——两者只差这一件事）；鼠标移动进入该UI所在的边缘区域（如顶部/底部一定高度的热区）才淡入显示；鼠标离开该区域后（可加小幅缓冲防抖动）立即淡出——**触发条件是"鼠标进入UI所在区域"，不是"画面上任意鼠标活动"**，因为这两种模式下核心操作（拖拽平移、滚轮缩放、以及自动平移本身）都需要持续的鼠标移动/画面运动，如果用"任意移动都唤醒UI"的常见视频播放器逻辑，UI会持续闪现，起不到沉浸效果
- **热区"武装"机制**（2026-07-10 定案）：刚切入卧游/神游时热区不立即生效——因为切换动作发生在顶部热区内，若立即响应，UI 退场后又马上被唤回，体验像 bug。必须先观察到鼠标离开过热区一次，该热区才开始响应进入事件
- **神游下比例 toast 照常出现**（2026-07-10 定案）：toast 是瞬时反馈不算常驻 UI，缩放时正常弹出并自动消失，不受"UI 全部淡出"规则约束

### 3.7 键盘快捷键
- `I` — 循环切换 读画 / 卧游 / 神游 三模式（或各分配一个键，实现时定）
- ~~`L` — 切换语言~~（随语言切换一起移出 MVP）
- `M` — 音乐开关
- `?` — 弹出全部快捷键速查浮层
- 所有快捷键在UI淡出、不可见状态下依然生效，不依赖鼠标先唤醒壳层——键盘路径应完全独立于鼠标可见性状态
- 顶部各按钮hover时展示tooltip，显示对应快捷键

### 3.8 标注 / 交互机制（核心功能，详细规格）

**视觉语言：朱笔圈点（单色朱红手绘圈），悬停时marker消失、原地被信息卡取代（不做常驻/悬停引线）**
- 每个可交互点位用一个**朱红色的手绘圈（圈点）**表示，带毛笔笔触质感 + Multiply 混合，读作"钤/画在绢上"而不是浮在屏幕上（放弃了早期的纯色矢量圆点方案，理由见 §6 与 `decisions-log.md`）
- **静止态一律单色朱红，不再用分类色区分**：静止态只负责传达"这里有东西可交互"，类别身份（建/人/物/木/兽）延后到悬停卡片里的字印上揭示——这条推翻了早期"marker 自带分类色"的设定，是消掉"彩虹散点"的关键
- **marker 按缩放门槛分层显隐，不再任何缩放都常驻**（详见下方"分层显隐"；这也把 §6 曾否决的 min_zoom 方案以"绑层级"的形式有条件重新引入）
- marker 尺寸：实物 100% 以内**固定屏幕像素**；超过后按 **√实物比例** 温和放大（封顶 3×，热区/聚合/计数字同步）。【2026-07-11 修订：原"任意缩放都固定屏幕像素"在 300%+ 实测圈异常小又细；√ 增速慢于画面，保留"放大后存在感递减"的方向、但不再看不清，仍不需要动态透明度淡化逻辑】
- 静止态 marker **无 idle 动画**（2026-07-11 取消呼吸动效——手绘笔触的天然不规则已提供足够的"非死板"感，动效反而让满屏圈点显得躁动）；"不死板"改由三种手绘圈随机分配实现（见下一条）
- **三种手绘圈变体随机分配**（按点位 id 稳定哈希，刷新不变）：让画面中的圈笔触各异，不是每个都长一样

**分类（可多选，共6类）：** 分类只是数据/筛选维度，**不再决定静止态 marker 的颜色**（静止态统一朱红）。下面这6个分类色现在只用在**悬停信息卡的字印**上（见"悬停交互"），实际 hex 见 §10（Figma 是 source of truth）：
| 分类 | 颜色 | Hex |
|---|---|---|
| 建筑 | 胭虹（暗红） | `#7C4449` |
| 人物 | 青蓝 | `#32788A` |
| 物件 | 灰蓝/石板 | `#6B798E` |
| 植物 | 灰绿 | `#6A8D52` |
| 动物 | 赭石/琥珀 | `#BE8A2F` |
| 印章 | 砖红/氧化红 | `#8A2E1E` |

（2026-07-11 新增"印章"类：指画上的鉴藏印/钤印——性质上是后人盖在画面上的痕迹/元数据，不同于前 5 类"画中描绘的内容"，但作为普通分类处理，**不单独成档**，tier / 分层显隐 / 聚合都与其它类一视同仁。）

一个点位可同时属于多个分类（tags数组），筛选逻辑 = 命中任一激活标签就显示。多标签用于"同一坐标点内容上确实横跨多个分类"的场景，不要用来表示"两个东西离得近但坐标不同"（那种情况应该分开建点，见下方"建筑与部件的标注惯例"）。

**层级（tier，管 marker 大小 / 内容深度 / 是否参与聚合 / 显隐门槛）：**
| 层级 | 定义 | 圈的大小 | 显隐门槛 | 聚合 |
|---|---|---|---|---|
| 地标层 | 全卷导航锚点级别的地标（如虹桥、孙羊正店、城门楼） | 大圈 | 缩小/总览时仍显示（当导航锚点），内容可写得更丰富 | 不参与 |
| 场景层 | 能用"动词+主语"描述的活动/情节（如说书人讲古、税吏查货） | 中圈 | 中等缩放开始浮现 | 参与 |
| 细节层 | 只能用"名词"描述的单一静态物件/人物（招牌文字、器物） | 小圈 | 到实物100%附近才出现 | 参与 |

**形态统一为"圈"，大小＝层级：** 三个层级都用手绘朱圈，只靠大小区分——地标=大圈、场景=中圈、细节=小圈（放弃了"细节用点、地标用印"的分形态想法）。大小是三档离散基准值（28/18/14 屏幕像素，实物 100% 以上整体乘 markerGrowth 温和放大；细节档 2026-07-11 由 12 调到 14 并加粗描边 0.7→0.9，实测 12 偏小），档间要留足间距、并压住手绘圈的天然大小抖动，避免"偏大的细节圈"和"偏小的场景圈"撞脸；如果三档实测分不清，在"仍是圈"的前提下用笔画粗细/双圈做次级区分，不退回点/印。

**圈落在目标上或旁边，不框住目标**（框住太复杂、画面元素边缘不好框）：偏移用固定屏幕像素、且优先落向目标靠空绢的一侧，配合悬停卡片标题，保证不含糊地指向被标注元素。

**分层显隐（缩放门槛，绑层级）：** 标识只在画面放大到**实物100%（即默认初始视图的尺度）及更近**时才显示，缩小到总览时隐藏——缩得太小时元素本就看不清，显示圈是多余信息，圈是配合"看清元素"才有意义。因为初始视图就是实物100%，实际效果＝"总览干净、进入阅读态就有圈"。门槛**也绑层级**：地标层缩小/总览时仍可见、继续当全卷导航锚点；场景层中等缩放开始浮现；细节层到实物100%附近才出现。过渡用一小段缩放区间淡入/晕开（约80%→100%），不在正好100%那一帧硬弹出。总览既要干净又要能寻路，就把"哪里有内容"的提示交给底部缩略图导航条去背，主画布保持纯净。（此条把 §6 曾否决的"一刀切 min_zoom"以绑层级、地标豁免的形式重新引入，绕开了当初的可发现性顾虑。）

**tier与分类（tags）是两个独立维度，互不绑定：** 聚合豁免是由tier（地标层）决定的，不是由分类（建筑）决定的——理论上一个建筑分类的点位不一定是地标层，一个地标层的点位也不一定是建筑分类。

**内容标注惯例（authoring convention，非schema硬约束）：** 这幅画里值得单独标"建筑"分类的点位实际上都够得上地标级别，因此约定——建筑分类的点位默认按地标层处理。建筑的具体部件（门、窗、招牌）不归进建筑分类，改标物件类，且**各自独立建点**，不要用多标签压在建筑本体那一个坐标点上——因为部件和建筑本体本来就是画面上两个不同的锚点坐标，分开建点也让两者各自拥有独立的标题/正文。这条约定不在schema或标注工具里做硬编码强制（不因为选了"建筑"就锁死tier），保留手动例外的灵活性。

**精度实现：**
- marker锚点坐标存储为真实完整卷轴分辨率（160348×7435px，已切金色画+上下裁边后的基准）下的content-space像素坐标，与工具/应用内部实际渲染用哪份分辨率的图片无关
- 渲染时用跟画布图块相同的pan/zoom变换矩阵计算marker锚点的屏幕位置——因为是单点坐标不是形状轮廓，不存在近似误差，任何缩放倍数下都精确贴合，不会漂移
- marker相对锚点的偏移量（不落在内容正中心，避免遮挡被标注元素本身）用固定屏幕像素定义，不随缩放放大

**悬停交互：**
- 悬停时marker本身**淡出/消失**，信息卡在其附近原地浮现——**不画连回锚点的引线**
- 卡片摆放硬规则：**绝不能遮住该点所标注的画面元素本身**（锚点周围留一圈"保护区"，具体判定算法/半径留到后续讨论，可能随tier变化）；**可以**遮住其他不相关的邻近marker/内容——这两条约束替代了引线，解决"密集区卡片对应哪个点"的辨认问题
- **marker视觉消失不等于交互移除**：点击热区、"点击=精确放大到锚点"这个行为在marker被卡片取代期间依然生效
- 信息卡内容 = **标题 + 正文**；正文用 Markdown 式约定承载引用块——**以 `>` 开头的行渲染成金色引用 block**（空行分段，可多段、可与正文穿插），其余是普通正文段；解析实现见 `annotator-tool.html` 的 `parseBlocks`（app 卡片端复用同一套，与行内链接共用 markdown 解析）。不做分类/层级 tag pill 展示，不做"查看更多"按钮。**正文/引用块内可有行内链接**，语法 `[显示文字](目标)`：目标为 `https://…` = 外链（新标签打开）、`#<点位id>` = 站内跳到另一个标注点（相机运镜过去并展开）；链接颜色统一用 UI红（§10），不随分类变化。**内容约定：金色引用块宜放与正文不重复的内容**（原始文献、题记、点睛引语等），别把正文原句再抄一遍——两段几乎一样会显得冗余
- **印章(seal)视觉**：信息卡上用一个汉字印章图标表示分类，颜色用该分类的分类色（§10那5个色）——**分类色如今只在这里用，静止态 marker 已经不带分类色了**。目前已定稿两枚："建"对应建筑、印章类用单字**「印」**（Figma 108:3808，2026-07-11 已实现——砖红 #8A2E1E 方印+纸面色字，SVG 字形由刻本宋字体轮廓生成，取代此前"「印章」二字"的说法）；人物/物件/植物/动物对应的字尚未最终拍板，等用户设计图定稿后更新（组件库当前以 人/物/木/兽 占位）
- **信息卡需要支持交互**：内容长时可内部滚动查看，内容里的链接可点击跳转。为此marker和卡片要共享同一个悬停状态（两者各自监听mouseenter/mouseleave，任一方在悬停中就视为整体在悬停），并在鼠标离开时加一个约250-300ms的缓冲延迟再判定隐藏——如果这段时间内鼠标成功移入了卡片，就取消隐藏。这样鼠标从marker移动到卡片之间的空隙不会导致卡片提前消失。如果这个缓冲延迟的方案在实测中不够用（比如卡片被自动避让弹得较远），可以升级成"安全三角形"（marker-鼠标-卡片三点围出的三角区域内移动都不判定为离开）这种更宽容的几何判定方式，作为后续优化，不是MVP必须
- **卡片本身的wheel事件必须stopPropagation**：卡片内部滚动查看长内容时，不能让这个滚动同时触发画布层的滚轮缩放
- **卡片本身的click事件必须stopPropagation**：点击卡片内的链接或任意内容，不能触发画布层的单击放大（见3.5）

**密集区域聚合（clustering）：**
- 判断标准是**当前屏幕像素距离**，不是缩放百分比——两点屏幕距离低于阈值就合并成聚合标记，超过阈值就拆成独立marker，这条规则天然适应不同区域的内容密度差异，不需要按区域手动调参
- 地标层不参与聚合，只有场景层和细节层互相之间参与聚合运算
- 双阈值防抖动（大致40-60px区间，具体数字**留到真实内容实测再定，不在MVP设计阶段拍死**）。附带一提：上面的"分层显隐"本身已经把密度稀释了大半，聚合逻辑大概率能做得更轻
- **聚合标记 = 一个"这里还有东西、放大看"的入口，不再展示悬停列表。** 理由：列表给的是抽象标题，而这套内容的意义全在画面里——看到真实场景之前，标题（如"税吏查货"）没有意义；列表把人从"看画"拽进"读目录"，还和"点击放大"功能重复
- 悬停聚合标记**只给一个轻提示**：光标变放大镜 + 首次进入的引导，尽量不常驻文字（常驻说明文字压在画上本身违和，且用户学会一次即冗余）
- **只有一条点击路径**：点击聚合标记 = 以它为中心相机运镜放大到超过裂散阈值，聚合裂成独立的圈；滚轮手动放大到该区域同样会裂开。聚合不是特殊模式，只是重叠的圈被临时折叠的形态，用任何方式凑近都会展开
- **裂开即一次性全放出、允许轻微重叠，不做多级次聚合**——用户已主动凑近，就不再设第二道关卡
- 有意识接受的代价：去掉列表也就失去"在密集区一步直达某个具体点"的能力。可接受——这是探索式浏览不是搜索，且抽象标题对这套内容没价值；定向跳转留给路线图里的主题筛选/逐项浏览
- **聚合标记样式**：**实心朱圈**（对比单点圈的空心，"填满＝内容多"，这个视觉逻辑自解释），中间用**汉字数字**显示聚合了几个（一…九），**超过九显示"众"**（"众"读作"这里很多"，不用数字也懂，留在画的字语言里）。聚合标记保持**单一固定尺寸**（不参与"大小＝层级"，数量交给中间的字），且不应比地标大圈更抢眼。颜色用朱红即可——既然静止态已全部单色，"聚合用中性色以免混进分类色"的旧理由已失效；质感上同样做"进绢"（像一枚朱印/一团朱砂晕、数字用留白负形透出），不要平涂徽章感

- 不需要登录，不做任何跨会话的观看历史记录（当前会话内的临时状态，如卷轴呈现方式的选择，属于正常应用状态，不算"历史记录"，刷新页面后重置为默认值）

### 3.12 卷首序页（Figma 108:3836 / 150:1183，2026-07-12 框架落地，同日改版：纸页形态+首次进入模式+宣纸纹理）

每次打开网页默认出现的序页，介绍画作背景后由用户主动「展阅」进入画作。**无持久化**——刷新即重置。

**形态：1225×807 纸页水平垂直居中，非全屏铺满**（初版误读 148:1233 为全屏，已纠正）。收缩规则分 variant（2026-07-12 用户拍板加呼吸边距）：**首次进入**与视口保持 **48px 边距**（`width: min(1225px, max(985px, 100vw−96px))`、`height: min(807px, 100vh−96px)`）——窗口缩小到触发最小视口门槛前始终留白；宽度方向 1081px 以下让步到最少保 985px 纸页宽（左列与中央列不重叠的下限，1024 视口实测 left.right 141.5 < center.left 149.5）。**重开 overlay** 不适用 48 边距，仍 `min(1225px, 100vw−40px)` / `min(807px, 100vh−40px)`。纸面 = `--paper-80` + **宣纸纹理**（Figma Paper texture 原图 4096²，压至 1536² 入库 `assets/intro/paper-texture.jpg`；multiply 进纸 + opacity 0.6 与 Figma 渲染目测校准；`isolation: isolate` 防纹理混合泄漏到纸页之外）。

**首次进入 vs 重开（variant，2026-07-12 用户拍板）：**
- **首次进入**（'first'）：backdrop 为 opaque 网页底图（AppBackground 渐变+莲花纹）——**不展示背后的画作**；壳层 UI（模式切换/分段导览/音乐/卷首按钮/导航条/比例簇/清晰度条）**全部隐藏**；**音乐不播放**（按钮不可达+无 autoplay）。画布仍挂载在底图之下**照常预载瓦片**——首次「展阅」揭开时画面即刻清晰。首次点「展阅」的瞬间壳层开始滑入（纸页+底图一起 400ms 淡出，variant 在整个 open→closing 周期内固定，保证平滑露出画作）
- **重开**（'overlay'，顶栏 `auto_stories` 按钮，图标为占位选择）：纸页直接 overlay 在画面上——四周画作可见、纸面 80% 透出画面，无首次逻辑；wrapper 拦截画布交互。**CTA 换「关闭」**（Figma 152:1270：UI金 `#DCCCA8` 底 + 字金 `#7F6019`，几何与「展阅」相同）；**点纸页以外的背景也可关闭**（2026-07-12 拍板，取代早先"不做点外关闭"；背景 cursor: pointer 作提示，点纸页本身不关）。首次进入不适用点外关闭——唯一出口是「展阅」

- **坐标系统**：真实完整卷轴分辨率（**160348×7435px**）为官方坐标系（**source of truth，已锁定**）。这版由故宫版原始扫描件`Full scroll/1-4.jpg`按右→左顺序拼接、切掉卷首金色装裱画、再上下裁掉扫描白/黑背景边（顶100/底60全分辨率px）后得到；官方master文件为项目根目录的`清明上河图-完整拼接.jpg`。应用内部无论用什么分辨率的素材渲染，标注坐标始终以此为准。旧的182054基准（含金色画）已作废——见§8

## 6. 已评估并放弃的方案

以下方案讨论后明确放弃，实现时不应重新引入，详细推理见`decisions-log.md`：

1. **轮廓贴合"背光"效果**（hover时给画面元素本身描边发光）——手描轮廓在不同缩放倍数下无法保持精确贴合，且684个点位的手描成本过高
2. **用min_zoom一刀切gate所有点位的"存在"**（不分层级，缩放不到阈值就全部不显示）——这个一刀切版本仍然放弃，因为会让用户彻底错过知识点，违背"学习模式要让人发现历史信息"的初衷。**但注意：一个"绑层级的分层显隐"已被采纳**（地标层缩小时仍显示当导航锚点，场景/细节层才随放大逐层浮现，见§3.8"分层显隐"）——它保留了"细节缩小时是噪音"的好处，又用地标豁免解决了可发现性，跟被否的"一刀切"是两回事
3. **缩小到某个百分比区间才显示indicator（一刀切版本）**——同上，一刀切版本放弃；现改为§3.8的"绑层级分层显隐"
4. **高倍缩放时marker动态透明度淡化+悬停唤醒**——决定不做这套动态逻辑，改用marker固定屏幕尺寸+低调静态视觉设计来覆盖同样的问题，保持逻辑简单
4b. **5个分类色直接给静止态marker上色**（早期设定）——放弃：五色高饱和圆点铺在退色绢本上＝彩虹散点，杂乱且与画面材质格格不入。改为静止态统一单色朱红的"朱笔圈点"，分类色只在悬停卡片的字印上出现（见§3.8）
4c. **用纯色矢量圆点当 marker**——放弃：干净的矢量圆读作"贴在屏幕上的现代UI"。改为带笔触质感 + Multiply 混合的手绘朱圈，读作"钤/画在绢上"
5. **悬停信息卡固定dock展示**——决定改用floating，理由见§3.8
6. **悬停时常驻画一条连回锚点的引线**——早期定案，后来被用户自己的proof of concept推翻：改为悬停时marker本身消失、卡片原地取代，靠"卡片不能遮住自己标注的锚点、但可以遮住其他不相关内容"这条硬规则解决辨认问题，不再需要线。详见`decisions-log.md`
7. **建筑分类天然不参与聚合**——不准确的表述，已澄清聚合豁免是tier（地标层）决定的，不是分类（建筑）决定的；"建筑默认按地标层处理"只是一条内容标注惯例，不是架构规则

## 7. MVP 之外（路线图，明确不在本期范围）

- **"放大到最大"按钮**——跳转到最高可用精度倍数的快捷缩放按钮，移出MVP，等深度缩放瓦片架构的最高精度层设计明确后再考虑加回来
- **方案B：next/prev逐项浏览组件**——把标注内容封装成独立UI，点击时相机自动运镜对齐每个点位，信息卡固定dock展示，列表内容随当前视野动态变化。作为读画模式主交互方式（floating悬停）之外的**补充**功能，不替代主交互模式。注：其中"自动运镜+固定dock卡片"的被动体验诉求，已由**卧游/神游的自动展卷**（§3.2）覆盖；剩下手动 next/prev 逐项浏览仍可作补充

- **主题线索跨卷筛选**——选中一个主题标签，全卷相关点位高亮并可跳转浏览
- **自动导览的旁白层**——被动式"自动展卷"本身已由**卧游/神游**落地（见§3.2）；这里剩下的是在其上叠加**旁白/语音讲解**高潮场景，属于内容+音频层的增强，仍在路线图
- **场景环境音**——随卷轴区域变化的环境声，讲解时环境音duck
- **URL深链分享**——将当前位置+缩放级别编码进URL，无状态分享
- **"?"快捷键速查浮层之外的完整设计系统**——正式的视觉token（色彩、字体、间距、动效时序）尚未系统化整理，属于视觉设计阶段的工作，不阻塞MVP功能范围的确定
