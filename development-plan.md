## Milestone 1 — 核心交互框架 ✅ 已完成

壳层交互与画布浏览体验，坐标系已按真实分辨率（160348×7595）搭好，`src/viewer/`。

- [x] 学习/沉浸模式切换：沉浸时顶部 UI 上退、底部 UI 下退（定稿：滑出+渐隐，1000ms easeOutQuint）
- [x] 沉浸模式边缘热区唤醒（顶 120px / 底 160px，离开 200ms 缓冲），触发条件是"鼠标进入热区"而非任意移动（PRD §3.6）
- [x] 热区"武装"机制：切入沉浸后须待鼠标离开过热区一次才生效，避免 UI 退场后立刻被唤回；沉浸模式下 toast 照常出现（均 2026-07-10 定案）
- [x] 拖拽平移（5-8px 移动容差，偏向判定为拖拽）
- [x] 单击放大 ×1.5（在当前比例上 +50%），锚定点击像素，250ms 缓动（无双击档）
- [x] 滚轮缩放，锚定鼠标位置
- [x] 缩放范围：最小 = 整卷完整入画（fit 全览，动态）、暂定最大 = 扫描分辨率 500%【2026-07-10 改版，废弃"锁最小 100% 扫描"】
- [x] 比例基准 = **实物比例**（100% = 画高 24.8cm 按 CSS 96dpi 折算的真实物理大小 ≈ 扫描 12.3%），初始视图 = 卷首 + 实物 100%
- [x] 比例 toast：缩放时出现，约 1.2s 后自动消失（时长常量 `TOAST_HIDE_MS`，待实测调整）
- [x] 1:1 还原按钮：不常驻，只随 toast 一起出现（且比例 ≠100% 时）；点击回到实物 100%，保持当前观看位置中心不变
- [x] 底部导航条：实时视口指示框（位置+大小随缩放联动），点击跳转（350ms 运镜）
- [ ]（遗留）指示框视觉样式为占位方案，等视觉稿定稿后替换

## Milestone 2 — 深度缩放瓦片（画质架构）

现在用 16000×758 工作副本渲染，高倍缩放是糊的。此里程碑替换底图管线，交互与坐标系不变。

- [ ] 瓦片切割脚本：从全分辨率原图（`Full scroll/1-4.jpg` 拼接）生成金字塔瓦片（DZI 或自定格式）
- [ ] 按视口 + 缩放级别流式加载瓦片，低清先行、高清渐进替换
- [ ] 加载进度指示：含义 = "当前视口清晰度"（PRD §3.1），不是整图进度
- [ ] 最高精度层确定后：评估"放大到最大"按钮是否加回（PRD §7）
- [ ] 性能验收：任意缩放/平移无明显卡顿，内存可控（瓦片卸载策略）

## Milestone 3 — 标注系统 MVP（核心功能）

与标注内容生产汇合。数据 schema 已定（PRD §4），坐标变换直接复用 viewer 的矩阵。

- [ ] 标注数据加载（annotator-tool 导出的 JSON）
- [ ] marker 渲染：content 坐标锚点 + screen 固定像素尺寸/偏移，跨缩放不漂移
- [ ] marker 分类色 × 层级尺寸（地标/场景/细节），呼吸/微光 idle 动效
- [ ] 悬停：marker 淡出、信息卡原地浮现（无引线）；marker 与卡片共享悬停态，250-300ms 离开缓冲
- [ ] 卡片"保护区"摆放规则：不遮住所标注的画面元素本身（具体算法届时与用户讨论，PRD §8）
- [ ] 卡片 wheel/click stopPropagation（不触发画布缩放）
- [ ] marker 点击 = 精确放大到锚点（复用单击放大逻辑）
- [ ] 沉浸模式下 marker 全部隐藏
- [ ] 学习模式首次进入的悬停机制引导提示（待定项，PRD §8）
