# CLAUDE.md — 清明上河图互动Web App

## 项目概述
让观者与高清《清明上河图》互动，通过预标注的画面元素学习北宋人文历史信息的网页应用。画面为主，UI服务于画面。

## 核心功能
1. 放大画作，仔细欣赏细节
2. 与预标注元素互动（建筑 / 人物 / 物件 / 植物 / 动物 / 印章）学习历史信息

## 关键规则
- **三种观看模式**（2026-07-11 由两模式扩为三，详见 PRD §3.2）：**读画**（原"学习"，主动+有信息，UI常驻可交互，默认）/ **卧游**（重新定义：画卷自动右→左缓慢平移+标记全在+壳层常驻；缩放不打断平移、悬停标记缓停读卡、移开缓起、可拖拽快进倒带；进入从当前位置/缩放接着走、不回卷首；卷尾停+常驻「重播」）/ **神游**（新，UI全隐纯欣赏：运动逻辑同卧游但无标记无壳层，靠边缘唤出退出）
  - **核心原则：镜头（位置+缩放）是持续共享层，切模式绝不搬动镜头**——只改"自动平移开不开、UI/标记显不显示"；回卷首只在首次加载或点「重播」时。卧游/神游共用一台 auto-pan 引擎 + 一个"显不显示 UI/标记"开关。代码代号 learn/immerse 需扩成三个（读画=learn，卧游/神游各一）
  - 放弃了"自动逐个弹开每张卡片"的导览式播放，改成卧游"匀速流过+悬停哪个读哪个"（保留用户选择权、不强制走完全部点）
- 卷轴呈现MVP只做**完全展开**（连续长卷，hotspot生效）；分段臂展格式+切换checkbox已移出MVP，见"非MVP路线图"
- 缩放范围【2026-07-10 改版，废弃旧"锁定最小100%扫描分辨率"规则】：最小 = **整卷完整入画**（fit 全览，随视口动态计算）；最大暂定扫描分辨率的 500%。比例读数与"1:1 恢复实际大小"都以**实物比例**为基准——100% = 画在屏幕上呈现真实物理大小（按画高 24.8cm 与 CSS 96dpi 折算，≈ 扫描分辨率的 12.3%），目的：让用户直观感受原作实际大小。初始视图 = 卷首 + 实物 100%
- 位置指示组件（缩略图导览条），**通栏铺满整屏**（不居中限宽），可点击跳转；缩放控制簇MVP只保留"恢复实际大小"一个按钮，"放大到最大"已移出MVP
- 背景音乐（`app/public/music/bgm.m4a`，源文件 `Music/music.mp4` 转码 AAC 128k）：**默认不播放**，off 态用 Google `music_off` 图标；首次点击从头播放（循环），再点暂停，再点从暂停处续播
- **卷首序页（Figma 108:3836 / 150:1183，2026-07-12 框架已落地）**：**1225×807 纸页居中，非全屏铺满**（首次进入与视口保持 **48px 呼吸边距**、宽度 1081 以下退让到保 985 纸页下限；重开 overlay 形态仍 `min(1225, 100vw−40)`），`--paper-80` 底 + **宣纸纹理**（Figma Paper texture 原图，multiply + 0.6 目测校准）——左列竖排瘦金体「清明上河图」64px+朱印（**固定纸页左侧**，标题与印为定稿不变；瘦金体已子集化成5字1.8KB，改标题须重跑子集见 `fonts.css`）；中央 725px 列**纸页内水平居中且内部左对齐**：4枚纸签tab（写在前面/时代背景/张择端/递藏过程，实心纸面底）+ 正文 18px/**行高2.2**（信息卡是2.0，别混）；**tab固定只滚正文**，滚动区上下渐隐同信息卡；底部 CTA「展阅」（UI红/白22Bold/字距3.3，上24下48）**永远压在正文之上**，点击淡出400ms关闭。**首次进入模式**（刷新重置、无持久化）：backdrop = opaque 网页底图（渐变+莲花纹）**盖住画作**，壳层 UI 全部隐藏、音乐不可达不播放，画布藏在底下照常预载瓦片；首次点「展阅」后壳层才登场。**首次进入有墨染入场（2026-07-12 四版·生长窗口）**：底图先独享 400ms 静场 → **单一墨渍连续洇开**：实色纸层带静态墨斑 mask（`assets/intro/ink-blob.png` 程序生成），动画 `transform: scale` 0.12→3.12（**纯色缩放不变**，肉眼只见墨形边缘生长；scale 3.06 与纸页尺寸无关地恰好盖满对角）；纹理层/内容各自独立淡入聚焦，总长 ~3s；**全程只动 transform/opacity/filter（合成器属性）、mask 只光栅化一次**——二版动画 mask-size 属主线程逐帧重光栅（尾段四角必然顿挫）、三版四层错峰淡入被读出"一波一波"，两条路都勿回头；纸面用预合成实色 `#DED5C7`（首次底图恒定，纸80%×莲花墙中值）；**内容不分批**、随纸页一起从墨里浮现（分批展示暂缓、待用户回头再定）；重开无仪式、普通 400ms 淡入；respects prefers-reduced-motion。之后顶栏 `auto_stories` 重开 = 纸页直接 overlay 在画面上（无 backdrop、无首次逻辑；图标为占位选择）；**重开时 CTA 换「关闭」**（152:1270，UI金底+字金，几何同「展阅」），且**点纸页以外的背景也可关闭**（背景光标变 pointer；首次进入不适用、只能走「展阅」）。tab文字与正文都是**占位**、只读 `app/introContent.json`（唯一源），终稿待用户。纸签视觉已提取为通用 `PaperTabs` 组件（SegmentNav/卷首共用）
- 只做网页（不适配手机，提示用大屏），平板暂不考虑；不做登录，不记录观看历史
- **最小视口 1024×640（2026-07-11 拍板）**：低于任一维度即停止渲染画布（瓦片请求随停），整屏显示提示卡（当前为 placeholder，最终设计稿待用户提供）；resize 实时进出、恢复不丢观看位置。与"移动端 UA 检测"是互补的两道检查。顶栏边距同日由 48 收窄到 32（上/左/右）
- 加载：不阻塞式整图加载，推荐深度缩放瓦片技术，进度条含义="当前视口清晰度"
- 底图：故宫博物院藏张择端原本；**英文支持与语言切换按钮移出 MVP（2026-07-11，下个版本再做）**——数据 schema 保留 `*_en` 字段继续填，UI 只出中文；原定"双语中英全局toggle（不并排）"设计不变，进"非MVP路线图"

## 标注交互机制（当前定案）
统一 indicator，**悬停时marker消失、原地被信息卡取代**（不做常驻引线）：
- 静止态：**朱红色手绘圈（朱笔圈点）**，毛笔笔触质感 + Multiply"进绢"（放弃了纯色矢量圆点、也放弃了彩色）；**统一单色朱红、不用分类色**；尺寸在实物100%以内为固定屏幕像素，超过后按√实物比例温和放大（封顶3×，2026-07-11 修"高倍缩放圈异常小又细"）；**大小＝层级**（地标大圈/场景中圈/细节小圈，三级都用圈，放弃点/印分形态）；**无idle动画**（2026-07-11取消呼吸），"不死板"靠三种手绘圈变体按id稳定随机分配。**按缩放门槛分层显隐**（见下方），不再任何缩放都常驻
- 悬停态：marker本身淡出/消失，信息卡在其附近浮现——**不用连线**。卡片摆放硬规则：绝不能遮住这个点所标注的画面元素本身（锚点周围留一圈"保护区"），但**可以**遮住其他不相关的邻近marker/内容。这两条约束足够解决"密集区卡片盖住哪个点"的辨认问题，不需要引线兜底
- marker虽然视觉上消失，**点击热区和"点击=精确放大到锚点"的行为依然生效**——消失只是视觉替换，不是移除交互
- 分类（可多选tags，共6类）：只是数据/筛选维度，**不再给静止marker上色**；这6个色现在只用于**悬停卡片的字印**。hex见PRD第10节：建筑=胭虹`#7C4449`、人物=青蓝`#32788A`、物件=灰蓝`#6B798E`、植物=灰绿`#6A8D52`、动物=赭石`#BE8A2F`、印章=砖红`#8A2E1E`（2026-07-11 新增；鉴藏印/钤印，作普通分类、不单独成档，tier/显隐/聚合与其它类一样）
- 层级（tier，管圈大小/内容深度/是否聚合/**显隐门槛**）：**地标层**=全卷导航锚点，大圈，不参与聚合，**缩小/总览时仍显示**（当锚点）；**场景层**=能用"动词+主语"描述的活动/情节（如说书、查货），中圈，**中等缩放开始浮现**；**细节层**=单一静态物件/人物（如一只碗、一件招牌），小圈，**到实物100%附近才出现**。层级不处理"这几个点是否属于同一叙事分组"
- **分层显隐**：标识只在放大到**实物100%（默认初始视图尺度）及更近**时显示，缩小到总览隐藏（缩太小时元素看不清，显示圈是多余信息；圈是配合"看清元素"才有意义）；门槛绑层级如上；过渡在约80%→100%淡入、不硬弹；总览寻路交给底部导览条。圈**落在目标上/旁、不框住**（元素边缘不好框），偏移小并偏向空绢一侧
- **tier与分类（tags）是两个正交维度**：地标层不聚合，是tier决定的，不是"建筑分类"决定的——原则上建筑可以不是地标、地标也不一定是建筑。但作为**内容标注惯例**（非schema硬约束）：建筑类点位默认按地标层处理，因为这幅画里值得单独标"建筑"的基本都是地标级别；建筑的部件（门、窗、招牌）不归进建筑分类，改标物件类，并且**各自独立建点**（不是在建筑这个点上多加一个tag）——因为门和建筑本身占的是两个不同的锚点坐标，分开建点也能让两者各自拥有独立的标题/正文
- 锚点用content-space像素坐标 + 和画布同一套变换矩阵渲染（精确不漂移）；marker偏移用screen-space固定像素；marker尺寸实物100%以内固定屏幕像素、超过后随`markerGrowth`（√实物比例、封顶3×）放大，热区/聚合同步
- 密集区域用**聚合**处理：按屏幕像素距离（不是缩放百分比）判断是否合并，地标层不参与，双阈值防抖动，具体阈值留待实测。**聚合标记＝一个"放大看"的入口，去掉悬停列表**（抽象标题在看到画面前没意义，还和放大功能重复）。悬停只给轻提示（光标变放大镜 + 首次引导），不常驻文字。**只有一条点击路径**：点聚合＝以它为中心放大到超过裂散阈值、裂成独立圈；滚轮手动放大到该区域同样裂开（聚合只是重叠圈的临时折叠态，不是特殊模式）。**裂开即一次性全放出、允许轻微重叠、不做多级次聚合**。样式：**实心朱圈 + 汉字计数**（一…九，超过九显示"众"），单一固定尺寸、不比地标抢眼、同样"进绢"质感。代价（有意识接受）：失去"密集区一步直达某点"，定向跳转留给路线图
- 高倍缩放**不做动态透明度淡化逻辑**（保持简单）：marker视觉设计本身偏低调；尺寸增速（√实物比例）慢于画面本身，仍保留"越放大相对存在感越弱"的方向，但不再小到看不清（2026-07-11 实测"纯固定屏幕尺寸"在300%+时圈异常小又细，遂改温和放大）
- 信息卡内容只有标题+正文（不做分类/层级tag pill展示，不做"查看更多"按钮），正文内可以有行内可点击链接
- 印章(seal)视觉：每个分类在信息卡上用一个汉字印章图标表示，颜色用该分类色（分类色如今**只在这里用**，静止marker已单色朱红）。目前只确认"建"对应建筑；人/物/木/兽等其余字待用户设计图定稿
- 一个点位可属于多个分类（tags数组），筛选 = 命中任一激活标签就显示

→ 为什么放弃"轮廓背光"方案、放弃"min_zoom门槛"方案、放弃固定dock方案、放弃常驻/悬停引线方案：见 `decisions-log.md`
