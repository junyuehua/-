# CLAUDE.md — 清明上河图互动Web App

## 项目概述
让观者与高清《清明上河图》互动，通过预标注的画面元素学习北宋人文历史信息的网页应用。画面为主，UI服务于画面。

## 核心功能
1. 放大画作，仔细欣赏细节
2. 与预标注元素互动（建筑 / 人物 / 物件 / 植物 / 动物）学习历史信息

## 关键规则
- 两种观看模式：沉浸观画（UI淡出）/ 学习模式（UI可见+可交互）
- 卷轴呈现MVP只做**完全展开**（连续长卷，hotspot生效）；分段臂展格式+切换checkbox已移出MVP，见"非MVP路线图"
- 缩放范围【2026-07-10 改版，废弃旧"锁定最小100%扫描分辨率"规则】：最小 = **整卷完整入画**（fit 全览，随视口动态计算）；最大暂定扫描分辨率的 500%。比例读数与"1:1 恢复实际大小"都以**实物比例**为基准——100% = 画在屏幕上呈现真实物理大小（按画高 24.8cm 与 CSS 96dpi 折算，≈ 扫描分辨率的 12.3%），目的：让用户直观感受原作实际大小。初始视图 = 卷首 + 实物 100%
- 位置指示组件（缩略图导览条），**通栏铺满整屏**（不居中限宽），可点击跳转；缩放控制簇MVP只保留"恢复实际大小"一个按钮，"放大到最大"已移出MVP
- 背景音乐，可关闭
- 只做网页（不适配手机，提示用大屏），平板暂不考虑；不做登录，不记录观看历史
- 加载：不阻塞式整图加载，推荐深度缩放瓦片技术，进度条含义="当前视口清晰度"
- 底图：故宫博物院藏张择端原本；双语中英，全局toggle切换（不并排）

## 标注交互机制（当前定案）
统一 indicator，**悬停时marker消失、原地被信息卡取代**（不做常驻引线）：
- 静止态：小圆点marker常驻显示（不因缩放消失，**尺寸固定屏幕像素，不随内容缩放**），颜色=分类，尺寸=重要性分级，带呼吸/微光的idle动效（不是死板静止的实心点）
- 悬停态：marker本身淡出/消失，信息卡在其附近浮现——**不用连线**。卡片摆放硬规则：绝不能遮住这个点所标注的画面元素本身（锚点周围留一圈"保护区"），但**可以**遮住其他不相关的邻近marker/内容。这两条约束足够解决"密集区卡片盖住哪个点"的辨认问题，不需要引线兜底
- marker虽然视觉上消失，**点击热区和"点击=精确放大到锚点"的行为依然生效**——消失只是视觉替换，不是移除交互
- 分类（可多选tags，共5类，实际hex见PRD第10节）：建筑=胭虹`#7C4449`、人物=青蓝`#32788A`、物件=灰蓝`#6B798E`、植物=灰绿`#6A8D52`、动物=赭石`#BE8A2F`
- 层级（tier）：**地标层**=全卷导航锚点，不参与聚合，任何缩放都独立显示；**场景层**=能用"动词+主语"描述的活动/情节（如说书、查货）；**细节层**=单一静态物件/人物，用"名词"描述即可（如一只碗、一件招牌）。层级只管尺寸/内容深度/是否参与聚合，不处理"这几个点是否属于同一叙事分组"
- **tier与分类（tags）是两个正交维度**：地标层不聚合，是tier决定的，不是"建筑分类"决定的——原则上建筑可以不是地标、地标也不一定是建筑。但作为**内容标注惯例**（非schema硬约束）：建筑类点位默认按地标层处理，因为这幅画里值得单独标"建筑"的基本都是地标级别；建筑的部件（门、窗、招牌）不归进建筑分类，改标物件类，并且**各自独立建点**（不是在建筑这个点上多加一个tag）——因为门和建筑本身占的是两个不同的锚点坐标，分开建点也能让两者各自拥有独立的标题/正文
- 锚点用content-space像素坐标 + 和画布同一套变换矩阵渲染（精确不漂移）；marker偏移和marker尺寸都用screen-space固定像素（不随缩放变化）
- 密集区域用**聚合**处理：按屏幕像素距离（不是缩放百分比）判断是否合并成聚合标记，地标层不参与聚合，双阈值防抖动。**聚合标记默认中性色**（不做成员分类色的混合/拼色），除非该聚合恰好全员同一分类才显示那个分类色；悬停展示小列表（每项带自己的分类色）。**两条独立的点击路径共存**：直接点聚合标记本身＝相机放大靠近这片区域直到自然拆开；悬停后点列表里的具体项＝精确放大到那一个点的锚点（复用marker点击的放大逻辑，目标缩放倍数要保证这个点会超过拆散阈值）。具体聚合像素阈值留到真实内容实测再定
- 高倍缩放**不做动态透明度淡化逻辑**（保持简单）：marker视觉设计本身偏低调，加上marker固定屏幕尺寸这条规则自带"越放大marker相对存在感越弱"的效果，两者叠加即可覆盖"高倍缩放时marker太抢镜"这个问题
- 信息卡内容只有标题+正文（不做分类/层级tag pill展示，不做"查看更多"按钮），正文内可以有行内可点击链接
- 印章(seal)视觉：每个分类在信息卡上用一个汉字印章图标表示，颜色跟随该分类的marker颜色（不是传统统一红色）。目前只确认"建"对应建筑；人/物/木/兽等其余字待用户设计图定稿
- 一个点位可属于多个分类（tags数组），筛选 = 命中任一激活标签就显示
